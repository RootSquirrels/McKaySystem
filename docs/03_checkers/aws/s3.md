# AWS S3 checker

Status: Canonical  
Last reviewed: 2026-02-15

**Source code:** `checks/aws/s3_storage.py`

## Purpose

Emit S3 governance and storage optimization signals at bucket scope.

## Checker identity

- `checker_id`: `aws.s3.storage`
- `spec`: `checks.aws.s3_storage:S3StorageChecker`
- `is_regional`: `False` (global checker run once per full scan)

## Check IDs emitted

- `aws.s3.governance.lifecycle.missing`
- `aws.s3.governance.encryption.missing`
- `aws.s3.governance.public.access.block.missing`
- `aws.s3.cost.bucket.storage.estimate`
- `aws.s3.cost.lifecycle.transition.review`
- `aws.s3.cost.multipart.upload.cleanup`
- `aws.s3.cost.replication.review`
- `aws.s3.cost.intelligent_tiering.review`

## Key signals

- Buckets missing lifecycle configuration.
- Buckets missing default encryption.
- Buckets missing full public-access-block controls.
- Storage-size cost estimate signal per bucket.
- Large Standard-heavy buckets without lifecycle transitions.
- Standard-heavy buckets now prefer one primary optimization path:
  high-object-count buckets lean toward Intelligent-Tiering, while lower-object-count buckets lean toward lifecycle transitions.
- Stale multipart uploads when no abort-incomplete-upload rule exists.
- Multipart cleanup findings now include a simple materiality band based on stale upload count and age, without extra API calls.
- Replication-enabled large buckets that may duplicate storage spend, with sharper focus on cold-data replication destination classes and multi-rule scope review.
- Large buckets with little or no Intelligent-Tiering usage detected.

- `pricing:GetProducts` (via pricing service)

## Determinism and limitations

- Bucket-level governance findings are deterministic for equivalent API responses.
- Storage estimates and optimization suggestions are directional and not a direct replacement for CUR.
- Tiering and lifecycle findings are heuristic reviews based on storage-class mix, lifecycle coverage, and object-count evidence.
- Multipart and replication reviews are strengthened using existing bucket data only; no extra CloudWatch or S3 round-trips are required.
- The checker avoids stacking both lifecycle-transition and Intelligent-Tiering reviews for the same Standard-heavy bucket when one recommendation is the clearer primary path.
- This checker intentionally emits multiple independent S3 signals for correlation/aggregation.
